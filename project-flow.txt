// Project overview
Frontend {
    Redux toolkit for state management

    Auth:{
        Login page
        Register page
    }
    
    Instructor {
        Dashboard page {
            card[ 3 Courses ]
            card[ 120 Students ]
            card[ 45 Lessons ]
        }
        MyCourses page {
            card [Title, Instructor Name, Short Description, edit button, delete button]
        }
        CreateCourse page [form]
        Settings page [change password, name]
    }

    Student {
        Dashboard page {
            card[ 5 Enrolled Courses ]
            card[ 2 Completed ]
            card[ 40% Avg Progress ]
        }
        BrowseCourses page {
            card [Title, Instructor Name, Short Description, Enroll Button]
        }
        MyCourses page {[Title, Progress Bar, Instructor name,  Continue Button]}
        Settings page [change password, name]
    }

    course page {
        if not enrolled {
          [ Title, Instructor Name ]
          [ Course Content (2 columns) ]
          Left: Description + Lessons
          Right{ 
            Instructor --> Card (edit, info)
            Student --> card (Enroll/ info)
          }
        } 

        if enrolled {
            [ Header ]
            [ Progress Bar 🔥 ]
            [ Main Content ]
            ├── Lessons List
            └── Action Card (Continue) --> open last video if not Completed
        }
    }

    lesson page [
        title 
        video 
        edit delete for Instructor
        completed for student
    ]

    course/[id]/learn {
        [ Sidebar - Lessons List ]
        [ Main Content - Video ]
        [ Bottom Controls - Next / Prev / Complete ]
    }
}

Backend {
    Server Actions --> call services(controllers in express js)
    Services --> to call database(controllers in express js)
    lib --> verifyAuth, prisma,
    api --> to put api handler for Oauth next
}
---------------------------------------------------------------------
1. System Overview
   Learning platform
   
   Roles: (Student, Instructor)

2. Core Domain Model
   - User
   - Course
   - Lesson
   - Enrollment (tracks progress)
   - LessonProgress (tracks completion)

3. Key Features
   - Authentication (Oauth email, password)
   - Course management ={
      - create course
      - edit course
      - delete course
      - add lessons
      - update lessons
   }
   - Enrollment system
   - Progress tracking

4. Architecture
   - Next.js App Router (Frontend + Server Actions)
   - Service layer (business logic)
   - Prisma ORM (data access layer)
   - PostgreSQL (database)

5. Authorization
   - Middleware for route protection
   - Role-based access control in server actions

6. Core Flow
   Instructor → Course creation → Lessons  
   Student → Enroll → Learn → Progress tracking


Answer in interview:
I started by defining the domain model and user roles, then designed the data relationships
, then built the system flow, and finally mapped it into Next.js architecture with server actions and service layer separation.
